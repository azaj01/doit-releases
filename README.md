## The shift

You're doing work that should be **doing itself.**

Every hour spent on research, drafts, organising, scheduling — an assistant could
handle that, quietly, in the background. Doit is how you hand it over. No
developer. No learning curve. Just space, given back to you.

**Before Doit**
- 3 hours researching competitors — twenty tabs, scattered notes, losing the thread.
- The same Monday update, every Monday — 45 minutes you'll never get back.
- Files and emails, slowly piling up.
- Scheduling, back and forth — four emails for something that takes thirty seconds.

**With Doit**
- A finished report, in twenty minutes — five competitors, the key insights, saved and ready.
- Drafted before you've had coffee — pulled from your week automatically.
- Everything in its place, always — by project, by date, set once.
- A confirmed slot on everyone's calendar — found, proposed, sent, followed up.

## How it works

**Three things. Then you're done.**

1. **Connect your tools** — One click each. No API keys, no setup screens. Email, Calendar, Files, Browser. Everything stays on your machine.
2. **Say what you need** — In your own words. No commands, no formats. *"Research our competitors"*, *"Draft the update"*.
3. **Walk away** — Watch it work if you're curious, or don't. Come back to a finished result, ready to use.

---

## Why DOIT

- **Plain-words commands.** Describe the outcome; DOIT drives the agent runtime.
- **Full agent surface.** Agents, chat (streaming + tool events + approvals), sessions, schedules, skills, files, knowledge, workboard, canvas, analytics, logs.
- **Local or remote.** Runs a bundled OpenClaw daemon on your machine, or connects to a remote gateway URL.
- **Native desktop.** Built with Flutter — Windows, macOS, Linux. Lightweight, always-on via system tray, with built-in auto-update on Windows.

## About this repository

This is the **public release repository** for DOIT. It hosts only the Windows
installer artifact (`doit.exe`); the application source is maintained in a
separate private repository. Each release is tagged `v<version>` and the latest
stable build is always available at `releases/latest/download/doit.exe`.
